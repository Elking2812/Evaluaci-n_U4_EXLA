# Evaluación de Conocimientos: GitHub y Markdown

 

## 1. Conceptos Generales

**¿Qué es un archivo Markdown y para qué sirve?**

> Un archivo Markdown es un tipo de archivo de texto que utiliza una sintaxis sencilla para dar formato al contenido, como títulos, listas, enlaces o texto en negrita. Sirve para crear documentos estructurados de forma rápida y fácil, especialmente en plataformas como GitHub.

 

**¿Cuál es el propósito del archivo README.md en un repositorio de GitHub?**

El archivo README.md sirve para proporcionar información importante sobre un proyecto, como su descripción, instrucciones de uso, instalación y otros detalles relevantes, ayudando a los usuarios a entender y utilizar el repositorio.

## 3. Flujo de Trabajo en GitHub Desktop

**¿Qué diferencia hay entre hacer un "Commit" y hacer un "Push"?**

* **Commit:** Commit: Es guardar los cambios realizados en el repositorio de forma local, incluyendo un mensaje que describe lo que se modificó.

* **Push:** Es enviar los commits guardados en el repositorio local hacia el repositorio remoto en GitHub para que otros puedan verlos.

 

**¿Para qué sirve el botón "Fetch Origin"?**

>sirve para verificar si hay cambios nuevos en el repositorio remoto sin descargarlos automáticamente, permitiendo ver si existen actualizaciones antes de integrarlas al repositorio local.
